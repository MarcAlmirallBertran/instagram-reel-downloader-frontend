## Architecture

This is a React 19 + Vite 8 SPA for downloading Instagram Reels — authenticated users submit reel URLs, the backend processes them asynchronously, and the frontend polls for status and provides download links.

**Stack:**
- React 19 with the React Compiler enabled (via `@vitejs/plugin-react`'s reactCompilerPreset + `@rolldown/plugin-babel`)
- Vite 8 for bundling and dev server
- TanStack Router v1 for client-side routing
- TanStack Query v5 for data fetching, caching, and polling
- Tailwind CSS v4 (`@tailwindcss/vite`) — theme tokens defined in `src/index.css` via `@theme {}`
- `sonner` for toast notifications
- `lucide-react` for icons
- `clsx` for conditional class names

**React Compiler note:** The React Compiler is active, so manual `useMemo`/`useCallback` optimizations are unnecessary. Avoid adding them unless there's a specific reason.

**ESLint:** Uses ESLint 9 flat config (`eslint.config.js`). Covers `.js` and `.jsx` files with `react-hooks` and `react-refresh` plugins. `no-unused-vars` is set to error but ignores variables matching `^[A-Z_]`.

## Routing

Routes are defined in `src/router.jsx` using TanStack Router's file-less approach:
- `/login`, `/register` — public, redirect to `/` if already authenticated
- `/` — dashboard (protected)
- `/tasks/$taskId` — task detail (protected)
- `/settings` — settings (protected)

Protected routes sit under an `authLayoutRoute` that calls `isAuthenticated()` in `beforeLoad` and throws a redirect to `/login` if the check fails.

## API

`src/lib/api.js` wraps `fetch` with JWT auth (token stored in `localStorage` as `access_token`). All requests go to `/api`, which Vite proxies to `http://localhost:8000` in dev (the `/api` prefix is stripped before forwarding via `rewrite: path => path.replace(/^\/api/, '')`). A 401 response clears the token and hard-navigates to `/login`.

**Media auth (Service Worker):** `<video>`/`<audio>` elements can't set custom headers, so `src/lib/auth-sw.js` registers a service worker (`public/auth-sw.js`) that intercepts fetch requests matching `/api/tasks/:id/files/(video|audio)` and injects the `Authorization: Bearer <token>` header before they hit the network.

**Auth endpoints:** `POST /users` (register), `POST /users/login` (form-data, returns JWT), `GET /users/me`, `PATCH /users/me`

**Task endpoints:** `GET /tasks`, `POST /tasks` (body: `{ uri }`), `GET /tasks/:id`, `POST /tasks/:id/cancel`, `GET /tasks/:id/files/video`, `GET /tasks/:id/files/audio`, `GET /tasks/:id/files/transcript`, `GET /tasks/:id/files` (zip download, via `tasksApi.downloadFiles()`)

## CSS / Theming

Tailwind v4 is configured via `@tailwindcss/vite`. Custom design tokens are declared in the `@theme {}` block in `src/index.css` (e.g. `--color-primary`, `--color-card`, `--color-border`). The palette is dark-only — there is no light mode.

**SVG icons:** Sprite sheet at `public/icons.svg`, referenced via `<use href="/icons.svg#icon-name">`.

## UI Components

`src/components/ui/` contains custom shadcn-inspired primitives: `Button`, `Card`, `Badge`, `Input`, `Select`, `Spinner`. They use `clsx` for conditional class names. These are **not** generated by the shadcn CLI and do **not** use `cva`.

## Task Statuses

Defined in `src/lib/status-config.js` and used by `task-card.jsx` and `task-detail.jsx`:

| Status | Label | Badge variant |
|---|---|---|
| `pending` | Pending | secondary |
| `in_progress` | Running | warning |
| `processing` | Running | warning |
| `completed` | Completed | success |
| `failed` | Failed | destructive |
| `cancelled` | Cancelled | secondary |

## Data Fetching Patterns

QueryClient defaults (set in `main.jsx`): `staleTime: 60s`, `retry: 1`.

- `useTasks(filters?)` accepts `{ status, sortBy, sortOrder }` — forwarded as query params to `GET /tasks`. Polls every 5 s when any task has `status === 'pending' | 'in_progress' | 'processing'`. Query key: `['tasks', { status, sortBy, sortOrder }]` — invalidating `['tasks']` prefix-matches all filter combinations.
- `useTask(taskId)` polls every 3 s while the individual task is active
- `useTranscript(taskId)` streams the response body via `ReadableStream` / `TextDecoder` (not a simple JSON fetch)
- Query keys: `['user']`, `['tasks', filters]`, `['task', taskId]`, `['transcript', taskId]`

## Docker / Deployment

The `Dockerfile` uses a two-stage build: `node:22-alpine` to build the Vite bundle, then `nginx:alpine` to serve it with `nginx.conf`.

`docker-compose.yml` maps host port `3000` → container port `80`. The container joins the external Docker network `instagram-reel-downloader` (must be created before `docker compose up`). nginx proxies `/api/` to `http://api:8000/` — the backend service must be named `api` on that network.
